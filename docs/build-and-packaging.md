# Build And Packaging

This project keeps one build graph in `Makefile` and derives both packaged bundles from it.

## Target Reference

| Target | What it builds or validates | Main outputs |
| --- | --- | --- |
| `make development-normal` | optimized development binaries | `build/development/normal/*` |
| `make development-debug` | debug development binaries (`-g -O0 -DDEBUG`) | `build/development/debug/*` |
| `make release` | internal optimized binaries used for packaging | `build/release/*` |
| `make test-suite` | runs 13 core geometry suites plus one independent terrain suite and the terrain JSON smoke | `build/development/normal/tester`, `build/development/normal/terrain_driver` |
| `make build/development/normal/terrain_driver` | builds the supported development-only terrain CLI | `build/development/normal/terrain_driver` |
| `make generate-build-info` | regenerates bridge provenance metadata | `tools/desmos-bridge/build-info.js` |
| `make release-bundle` | runs tests, builds release binaries, generates build info, assembles stable bundle | `dist/release/` |
| `make release-smoke` | rebuilds and validates the stable bundle | `dist/release/` plus smoke-test artifacts |
| `make interview-release` | runs tests, builds release binaries, generates build info, assembles interview bundle, seeds exports | `dist/interview/` |
| `make interview-smoke` | rebuilds and validates the interview bundle | `dist/interview/` plus smoke-test artifacts |
| `make clean` | removes build and packaged output trees | `build/`, `dist/release/`, `dist/interview/` |

## Build Profiles

| Profile | Compiler flags | Executables |
| --- | --- | --- |
| `development-normal` | `-Wall -Wextra -std=c++17 -O2` | release set plus `delaunay_driver`, `portfolio_export_driver`, `benchmark_driver`, and `terrain_driver` |
| `development-debug` | `-Wall -Wextra -std=c++17 -g -O0 -DDEBUG` | same set as `development-normal` |
| `release` | `-Wall -Wextra -std=c++17 -O2` | `main`, `interview_demo`, `rand_poly_gen_driver`, `driver`, `tester` |

`delaunay_driver`, `portfolio_export_driver`, `benchmark_driver`, and `terrain_driver` are
intentionally development-only. Terrain analysis is not copied into either
packaged bundle. The bundles ship a single primary executable each.

## Bundle Layouts

### Stable release bundle

Generated by `make release-bundle` into `dist/release/`.

| Path | Purpose |
| --- | --- |
| `dist/release/bin/planar-geometry` | packaged stable app built from `src/main.cc` |
| `dist/release/run.sh` | launcher that `cd`s to the bundle root before exec |
| `dist/release/QUICKSTART.md` | copied quickstart for the packaged polygon workflow |
| `dist/release/tools/desmos-bridge/index.html` | browser bridge for polygon and triangulation exports |
| `dist/release/tools/desmos-bridge/build-info.js` | generated provenance payload for the bridge badge |

### Interview bundle

Generated by `make interview-release` into `dist/interview/`.

| Path | Purpose |
| --- | --- |
| `dist/interview/bin/planar-geometry-demo` | packaged interview app built from `src/interview_main.cc` |
| `dist/interview/run-demo.sh` | launcher that `cd`s to the bundle root before exec |
| `dist/interview/QUICKSTART.md` | copied quickstart for the interview workflow |
| `dist/interview/examples/interview-demo-polygon.txt` | deterministic sample polygon copied into the bundle |
| `dist/interview/tools/desmos-bridge/index.html` | local visualizer shipped with the interview bundle |
| `dist/interview/tools/desmos-bridge/build-info.js` | generated provenance payload for the bridge badge |
| `dist/interview/tools/desmos-bridge/polygon-export.json` | seeded polygon export produced during packaging |
| `dist/interview/tools/desmos-bridge/triangulation-export.json` | seeded triangulation export produced during packaging |

`make interview-release` finishes by running `./bin/planar-geometry-demo --run-sample-demo --no-browser-launch` inside `dist/interview/` so the bundle ships with seeded exports.

## Build Provenance

The `Makefile` injects build metadata into compiled binaries through `BUILD_INFO_CPPFLAGS`:

- `GEOM_BUILD_COMMIT`
- `GEOM_BUILD_BRANCH`
- `GEOM_BUILD_TIME_UTC`
- `GEOM_BUILD_DIRTY`
- `GEOM_BUILD_PROFILE`

`scripts/generate_build_info.sh` writes matching metadata to `tools/desmos-bridge/build-info.js`, which is then copied into packaged bundles. `make release-bundle` and `make interview-release` run this automatically.

## Packaging Support Files

| Path | Role in packaging |
| --- | --- |
| `Makefile` | defines executable groups, build templates, packaging steps, and cleanup targets |
| `packaging/release/run.sh` | wrapper used as `dist/release/run.sh` |
| `packaging/interview/run-demo.sh` | wrapper used as `dist/interview/run-demo.sh` |
| `packaging/release/QUICKSTART.md` | source quickstart copied into the stable bundle |
| `packaging/interview/QUICKSTART.md` | source quickstart copied into the interview bundle |
| `scripts/generate_build_info.sh` | produces `build-info.js` for the bridge UI |
| `scripts/release_smoke.sh` | asserts bundle contents, runs the packaged stable app, and verifies generated exports |
| `scripts/interview_smoke.sh` | asserts bundle contents, runs the packaged interview app, and verifies seeded exports |
| `examples/interview-demo-polygon.txt` | source geometry copied into the interview bundle |

## Extending The Build

When adding a new executable to the current build system:

1. Add its name to the appropriate executable group in `Makefile`:
   - `COMMON_EXES` for binaries that should be built in development and release profiles
   - `DEV_ONLY_EXES` for development-only binaries
   - `PACKAGING_EXES` for release-built binaries that support packaging flows
2. Define `EXE_<name>_SRCS` with the entry source file list.
3. Add any shared implementation files to `COMMON_SRCS` if they should be linked into multiple executables.

If the new executable belongs in a packaged bundle, update the relevant packaging recipe in `Makefile` and the supporting quickstart docs under `packaging/`.
